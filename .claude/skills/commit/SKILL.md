# Git Commit

- Conventional commits: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`, `ci:`
  - `feat`: ユーザーにとって違いが分かる新機能を実装した時
  - `fix`: バグを修正した時
  - `refactor`: ユーザーにとって機能に違いはないがコードの書き方を変更した時
  - `docs`: コード内のコメントやドキュメントを書いた時
  - `test`: テストの追加、修正を行った時
  - `chore`: 依存関係を追加したや開発にだけ関係のある変更をした場合
  - `ci`: CI/CDに関係する変更をした時
- コミットメッセージは簡潔に保つ（1〜2文程度）
